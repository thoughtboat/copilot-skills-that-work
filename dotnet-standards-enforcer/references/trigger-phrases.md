# Trigger Phrases Reference

Example phrases that should invoke the `dotnet-standards-enforcer` skill, grouped by intent.
Used for documentation and training purposes; not loaded at runtime.

## Review / Audit / Enforce

- "review my C# code"
- "audit this project"
- "enforce coding standards"
- "check my code against our standards"
- "does this follow best practices?"
- "review this for code quality"

## Fix / Clean Up

- "clean up my code"
- "clean up this class"
- "fix style issues"
- "fix code quality issues"
- "tidy up this file"
- "refactor this to meet standards"
- "make this code production-ready"

## Apply Standards / Best Practices

- "apply coding standards"
- "apply .NET best practices"
- "improve this code"
- "make this idiomatic C#"
- "bring this up to standard"

## PR / Code Review Preparation

- "get this PR-ready"
- "prepare for code review"
- "review before I merge"

## Onboarding a New Codebase

- "apply our standards to this new project"
- "standardise this codebase"

## Sub-Skill Areas

- "add XML docs" → Documentation
- "fix async patterns" → Async Patterns
- "check DI registration" → DI & Services
- "check error handling" → Error Handling
- "fix configuration" → Configuration
- "fix localization strings" → Localization
- "check architecture" → Architecture
- "fix code quality" → Code Quality
